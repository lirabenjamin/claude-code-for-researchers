### 3. "Cryptographic teeth" — explanation + fix applied

By "cryptographic teeth" I was trying to say that **a git commit hash is a cryptographic hash** (SHA-1 of the full repo tree — every file, every byte). Two consequences:

- If anyone changes a single byte of your materials or code, the hash is different. You cannot fake it.
- If your dataset says "generated by commit `abc123`", and the code at `abc123` produces a different result, that discrepancy is provable. The link between dataset and code is **tamper-evident**.

That's what I meant by "teeth" — the phrase means "this has real enforcement power" in an informal register. The claim stands; the phrasing is jargony for a Wharton audience.

**I've replaced it in `demo_script.md`.** The Q&A answer now says "tamper-evident" and explains it in one line: "if anyone changes a single byte of the materials or code, the hash changes, and the link breaks." Same idea, no jargon. Check the updated §Katie Q1 in `demo_script.md`.
