Claude Code ≠ Claude Chat

Four differences worth naming:

  1. File manipulation — directly reads/edits files on your machine, instead of copy-pasting in and out
  2. Code execution + iteration loop — runs, catches its own errors, fixes, retries. Can work for 15-20 min on one task.
  3. Multi-agent — spawns sub-agents for parallel work under a coordinating Claude
  4. Long-term memory via markdown — the one that really changes the game

Long-term memory is the unlock

Claude Chat forgets everything when you close the tab. Claude Code doesn’t.

Four markdown files turn Claude into a collaborator:

  • CLAUDE.md — who you are and how you work
  • comms.md — async channel, for questions too big to answer inline
  • todo.md — running task list Claude updates as you work
  • current_state.md — handoff doc for cold-starting the next session

Unlike a human collaborator, this memory is inspectable and editable. Open the files. See what Claude remembers. Rewrite it.

What a SKILL.md looks like

Just markdown with a YAML header. Claude reads the description and decides when to invoke it.

---
name: pipeline-audit
description: Audit a research data-code-output-manuscript pipeline
  for correctness and consistency. Use whenever the user wants to
  verify their analysis pipeline, check for bugs in statistical code,
  find hardcoded numbers in a manuscript, or ensure narrative
  descriptions match the actual results.
---

# Research Pipeline Audit

Performs a comprehensive audit of a research project to ensure the
data → code → output → manuscript pipeline is correct, clean, and
internally consistent.

## The Four Audit Tasks

Run all four in sequence. Report findings for each before moving on.

### Task 1: Statistical Method Correctness

Goal: verify that the code actually implements what it claims to.

- Read all analysis scripts. Understand what each function does.
- For every statistical method used, verify:
  - The model formula is correct for the stated estimand
  - Fixed effects, weights, and clustering are applied correctly
  - Pre/post and treatment/control splits are defined correctly
  ...

That’s it. No magic.
